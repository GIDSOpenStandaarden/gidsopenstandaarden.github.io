Afsprakenstelsel : Release notes Launch 0.2

Created by Remy Lamers, last modified on mei 24, 2019

Release 0.2 ondersteunt functioneel de onderstaande functionaliteiten.

Use-case 1: Als persoon wil ik vanuit een wijkplatform drempelloos (zonder account aanmaken) naar een e-health module uit een e-health platform navigeren

Acceptatiecriteria: 

  1. Persoon navigeert tussen wijkplatform en zorgplatform, zonder nogmaals in te hoeven loggen
  2. Persoon kan een specifieke zelfzorg-module openen zonder in te hoeven loggen
  3. Persoon ziet dat het openen van zelfzorg-module in een nieuw tab gebeurt, bv door het toevoegen van een icoon
  4. Indien vereist kan de zelfzorg-module om aanvullende gegevens vragen.
  5. Persoon krijgt toegang tot e-health platform zonder dat persoon hier vooraf een account voor heeft aangemaakt.
    1. Het aanmaken van een account gebeurt onder water, bij het uitvoeren van de launch. De gebruiker geeft hiervoor consent (Zie use-case 2)


Toegevoegd t.o.v. 0.1

Use-case 2: Als persoon wil ik bij het openen van een zelfzorg-module via het wijkportaal, voor de eerste keer een scherm krijgen dat mij informeert welke gegevens gedeeld worden met het zorgportaal.

Acceptatiecriteria: 

  1. Persoon ziet de eerste keer dat een zelfzorg-module van een zorgportaal waar persoon nog niet bekend is, welke gegevens van het wijkportaal aan het zorgportaal verstrekt worden.

    1. Indien gebruiker accepteert, maakt ontvangende applicatie onderwater een account aan voor de gebruiker.

  2. Persoon kan aangeven dat het bericht gelezen is, hierna zal het bericht niet nogmaals verschijnen bij het uitvoeren van een launch

  3. Wanneer het persoon het bericht verborgen heeft, en launcht naar een applicatie waar gebruiker niet bekend is, ontvangt gebruiker wederom het consent bericht.

Wijziging

The form-post-redirect message

In step 5) in the communication protocol, the user is redirected to the producer with the JWT message using a post form. The form needs to:

  • Forward the JWT token in the request field of a form.
  • The page needs to request consent to the user and inform the user about the information being shared.

Appendix D contains a reference implementation of such a page.
